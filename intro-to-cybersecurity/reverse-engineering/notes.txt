--- A program ---

	- consists of modules
		- that are made of functions
			- that contain blocks of instructions

functions often begin with a prologue where the stack frame is set up and then end with an epilogue where
the stack frame is torn down

------

--- The stack ---

Contains local variables/env vars/program args

grows "backwards"

push -> rsp decreased by 8
pop  -> rsp increased by 8

When a function is called, the CPU automatically places the return address on the stack.

When the function finishes, that address is automatically removed from the stack so execution can continue 
where the function was originally called.

Every function sets up its stack frame:

	stack pointer (rsp) : 
	base pointer (rbp)  : 

Prologue : Save off the callers base pointer , set current stack pointer as base pointer , allocate space on stack
(subtract from stack pointer)

Epilogue : deallocate the stack (mov rsp, rbp) , restore old base pointer
